# LINUX DO Apple UI Roadmap

## 目标

在不增加接口请求、不影响 Discourse MessageBus、不隐藏或重排主题数据的前提下，
使用现有 DOM、站点 CSS 变量和已加载模型，将 LINUX DO 调整为更克制、清晰、
具有 Apple 桌面设计语言的界面。

设计原则：

- 内容优先，减少表格感和高对比边框。
- 毛玻璃只用于顶部导航、侧边栏和高架弹层。
- 主题列表采用 Inset Grouped 风格，不把每一行做成独立大卡片。
- 阅读页保持连续内容流，楼中楼保持紧凑层级。
- 所有样式同时兼容站点浅色与深色 CSS 变量。
- 动态增强必须幂等，空闲状态不得持续修改 DOM。

## 性能红线

后续阶段仍必须遵守：

- 不主动调用 MessageBus `poll`。
- 不增加定时器轮询页面状态。
- 不为每条主题或每个用户单独请求接口。
- 不在每个列表行使用 `backdrop-filter`。
- 不持续删除并重建相同节点。
- 不通过隐藏或重排大量主题行实现筛选。
- 不改变站点原有路由、键盘行为和无障碍属性。

## 后续建议

第三阶段应先从通知面板和用户菜单开始，这两个区域最适合使用高架材质；编辑器
改动范围较大，应单独进行视觉验证。第四阶段必须在第三阶段稳定后实施。
